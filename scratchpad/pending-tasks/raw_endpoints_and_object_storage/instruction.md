Encore's standard APIs enforce strict JSON validation, which creates friction when handling multipart form data like file uploads.

You need to implement a file upload system using an Encore `raw` API endpoint and a multipart parser (like `busboy`) to extract an uploaded file, storing the resulting file content in Encore's Object Storage.

**Constraints:**
- Define the API endpoint using `api.raw` to bypass standard JSON schema validation and gain access to the underlying Node.js request stream.
- Provision the storage infrastructure using Encore's native `Bucket` primitive.
- The endpoint must successfully stream the parsed file to the bucket and return the unique file identifier generated by the storage bucket as a plain text response.
